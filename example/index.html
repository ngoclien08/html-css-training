• width=device-width tells the page to match the screen's width in device-independent pixels (DIP). This lets the page reflow content to match different screen sizes.
  • DIP: a standard visual pixel unit (which can be made up of many physical pixels on a high-density screen)
  • Because some browsers keep the page's width constant when rotating to landscape mode, and zoom to fill the screen instead of reflowing
  • initial-scale=1 tells browsers to set a 1:1 relationship between CSS pixels and device-independent pixels regardless of device orientation, letting the page take advantage of the full landscape width.
  • Caution:
    • To ensure that older browsers can properly parse the attributes, use a comma to separate attributes: content="width=device-width, initial-scale=1.0"
    • In addition to initial-scale, the minimum-scale, maximum-scale, and user-scalable attributes are also available. They are not recommended because they can prevent the user from zooming the viewport, potentially causing accessibility issues.
